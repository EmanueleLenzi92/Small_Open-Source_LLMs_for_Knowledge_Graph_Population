# Small Open-Source LLMs for Knowledge Graph Population

## Overview
This project investigates the use of **small open-source Large Language Models (LLMs)** for **entity extraction and linking** from textual data, with the goal of supporting the **automatic population of Knowledge Graphs** in a real-world scenario.

The extracted entities are linked to **Wikidata** identifiers in order to obtain structured, reusable knowledge.

As a baseline, the project uses **ReLik**, a state-of-the-art system for **entity linking**, to compare the performance of the tested LLMs.

The real-world use case is based on data provided by the **Horizon 2020 MOVING project** <https://www.moving-h2020.eu/>, specifically a set of textual descriptions of selected european mountain value chains.

---

## Objectives
The main goals of the project are:

- evaluate the performance of small open-source LLMs on entity extraction tasks
- compare their results against a strong baseline (**ReLik**)
- assess how suitable lightweight LLMs are for real-world **Knowledge Graph population**
- enrich extracted entities with **Wikidata QIDs** for downstream structured knowledge applications

### `Results/`
Contains the outputs produced during the experiments.

#### `Results/LLMs_outputs/`
Contains the **raw JSON outputs** generated by the tested LLMs.

#### `Results/Results_with_Wikidata_QIDs/`
Contains the JSON files with the outputs of both the **LLMs** and **ReLik**, enriched with the corresponding **Wikidata QIDs** for each extracted entity, retrieved through the **Wikipedia APIs**.

### `dataset/`
Contains the **30 textual descriptions** of 30 selected value chains from the **MOVING project**.

### `goldStandard/`
Contains the **gold standard annotations** for each of the 30 MOVING files, including:

- the entities as they appear in the texts
- the corresponding **Wikidata QIDs**

## Experimental Pipeline
The workflow implemented in this repository can be summarized as follows:

1. textual data from the MOVING dataset are processed
2. multiple small open-source LLMs are prompted to extract named entities
3. the extracted entities are associated with Wikipedia titles
4. Wikipedia titles are converted into **Wikidata QIDs**
5. results are compared against the gold standard
6. performance is evaluated against the **ReLik** baseline

---

## Models
The project experiments with several small open-source models executed locally through **Ollama**, including models from families such as:

- Phi
- Gemma
- Llama
- Mistral
- DeepSeek

The focus is on lightweight models that are easier to run locally while still being potentially useful for practical Knowledge Graph population tasks.

## Python Scripts

### `ollama.py`
This script runs the extraction pipeline using multiple LLMs through **Ollama**.

Main functionalities:

- iterates over the dataset files
- prompts each model to retrieve named entities of type:
  - Person
  - Location
  - GPE
  - Organization
- asks the model to return the output in JSON format
- parses and cleans model outputs
- logs malformed JSON cases
- saves the extracted entities in JSON files

In short, this is the main script for generating the **raw LLM outputs**.

---

### `getWidataIdUsingWikipediaAPIs.py`
This script enriches the outputs by retrieving the corresponding **Wikidata IDs** from **Wikipedia titles**.

Main functionalities:

- reads JSON outputs produced by LLMs
- extracts the predicted Wikipedia title for each entity
- queries the Wikipedia API
- retrieves the related **Wikidata QID**
- saves the enriched results into new JSON files
- preserves the folder structure during batch processing
- skips already processed files

This script is used to transform raw extraction outputs into **entity linking results with Wikidata identifiers**.

---

### `relik.py`
This script runs **ReLik** as the baseline system.

Main functionalities:

- loads the pretrained model `sapienzanlp/relik-entity-linking-base`
- processes the dataset sentence by sentence
- extracts entity spans
- retrieves Wikipedia titles and Wikidata IDs
- saves the output in JSON format

This serves as the **reference baseline** for comparing the performance of the tested LLMs.

---

### `evaluation.py`
This script evaluates the results produced by the models and the baseline.

Main functionalities:

- loads gold standard and predicted JSON files
- computes **Precision**, **Recall**, and **F1-score**
- matches extracted entities using **Jaccard similarity**
- evaluates entity linking quality based on **Wikidata IDs**
- ranks models by performance
- identifies the best Jaccard threshold
- plots metric trends

This is the main analysis script for comparing approaches and understanding model behavior.

---

## Evaluation Methodology

### Matching
Predicted entities are matched against the gold standard using **Jaccard similarity** between strings.

### Metrics
The evaluation focuses on **entity linking** performance through:

- **True Positives (TP)**: correct entity mention and correct Wikidata ID
- **False Positives (FP)**: wrong entity or wrong Wikidata ID
- **False Negatives (FN)**: missed entities or entities linked incorrectly

The script computes:

- Precision
- Recall
- F1-score

It also supports analysis across different Jaccard thresholds.

## How to Run

### 1. Generate raw outputs with the LLMs
```bash
python ollama.py
```

### 2. Run the ReLik baseline
```bash
python relik.py
```

### 3. Enrich outputs with Wikidata QIDs
```bash
python getWidataIdUsingWikipediaAPIs.py
```

### 4. Evaluate results
```bash
python evaluation.py
```
